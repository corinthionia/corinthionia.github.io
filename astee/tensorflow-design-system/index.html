[DEVIEW2021] TensorFlow로 디자인 시스템 만들기
April 06, 2023

Motivation

1. 디자인 라이브러리 학습의 문제점

  • 디자인 라이브러리(디자인 시스템)가 점점 커질수록 프로덕트 디자이너들이 학습해야 할 게 늘어남
  • 여러 디자인 도구들도 이러한 문제를 해결하기 위해 노력 중인데, 그래서 그런지 작업 환경이 자주 바뀜
  • 어제의 최신 라이브러리가 오늘의 레거시가 되는 일도 발생…

2. 수작업의 한계

  • 결국 컴포넌트도 계속 늘어나고 새로운 기능(디자인툴)도 늘어나는데, 이 모든 걸 사람의 눈과 손으로 비교하여 맞추기에는 한계가 있음
  • 디자인 시스템은 점점 좋아지고 있는데 쓰기가 불편한 아이러니가 발생
  • 디자인 팀에서 리뷰도 하고 교육도 하는 방법도 있는데 이것도 사람이 하는 방식이니 한계가 있음

→ 디자인 시스템이라는 게, 자동으로 안 될까?
→ 컴퓨터비전 기술을 사용해서 화면을 자동으로 분석할 수 있다면, 디자인 맞춤법 검사도 가능하지 않을까?
→ 컴퓨터비전으로 화면을 분석하고, 디자인 시스템의 맞춤법을 확인해서, 오류를 자동으로 찾아준다면?

Prepare the data

1. UI 화면을 검출해야 했기 때문에 충분한 양의 고품질 UI 이미지 확보

  • Mobbin - 12,000장의 스크린샷 / 14개 카테고리로 라벨링
  • RICO - 93,000+ 안드로이드 스크린샷 / 27개 카테고리로 라벨링

2. RICO 데이터셋 가공하기

  • 원본 이미지, Annotation json을 다운로드
  • 이미지, annotation 파일만 각각의 폴더에 옮기기
  • Python 스크립트(convert.py)를 만들어서 Pascal VOC 포멧의 XML로 변환

3. UI 컴포넌트 정의 및 데이터 라벨링

  • Tensorflow용 TFrecord로 변환

Training and validating the model

학습 방법의 이슈

1. TensorFlow를 선택한 이유

  • 많은 것을 시도해 봤는데, 결과적으로 예제가 충분하면서 설치하는 데 돈이 들지 않고, 여러 플랫폼에서 쉽게 통합할 수 있는 Tensorflow를 이용하기로 결정
  • 단순 맞춤법 검사여서 누구나 설치 없이 바로 사용할 수 있도록 Tensorflow 중에서도 Tensorflow.js 기반의 웹 서비스를 만들기로 결정

2. 학습 방법이 결정되었으니, 거기에 맞게 데이터를 변환해서 업로드 해야 함

  • 앞선 작업의 결과물로 확보한 XML과 이미지 파일을 tensorflow에서 학습할 수 있게 묶어서 .tfrecord 파일로 변환해야 함 (이 과정에서 데이터셋에 문제가 없는지 여러번 검증 필요)

3. (선택) 이미지 사이즈 줄이기

  • 리코데이터셋의 원본 이미지 파일의 크기가 꽤 크기 때문에 나중에 학습할 때 원본 이미지를 그대로 쓰면 메모리를 과하게 사용하게 되는 이슈 발생

  • 이미지를 1/4로 줄여서 사용했을 때 훨씬 효율이 좋았음


이렇게 만들어진 TFRecord 파일(train, test) 파일을 직접 구글 드라이브에 업로드 하면 데이터 준비 끝!!


Backbone의 성능 및 선택 이슈

  • 모델을 바닥부터 만드는 일은 만드는 것도 어렵고 시간이 오래 걸리기 때문에 기존에 학습된 모델을 transfer learning 및 fine tuning 방법을 사용
  • 딥러닝 모델 자체는 tensorflow hub에 검색하면 정말 많이 나옴 (처음 공부할 때 이 부분 설명이 제일 어려웠음 ㅠㅠ)
  • 또 어려웠던 건 어떤 모델을 사용해야 하는지 정답이 있는 게 아니라, 주어진 상황에 적합한 모델을 선택해야 해서 더 난감했음

따라서 아래의 기준으로 모델 선택하고 몇 개의 모델을 성능 비교

  • ImageNet 성능 기준으로 좋은 딥러닝 모델을 선택
  • 웹 브라우저에서 동작해야 하므로 가벼운 모델을 선택
  • Finetune 성능이 차이나는 경우도 있으므로 실험 필요
  • ResNet50, efficientdet, MobileNetV2로 실험

MobileNetV2가 성능과 속도를 적절하게 만족! 이 모델을 backbone 모델로써 transfer learning 및 fine tuning을 준비함


트레이닝 이슈

데이터도 준비되었고, training 설정도 완료되었으니 이제 training과 validation을 진행하면 됨

  • 코랩 환경을 고려하여 7,500개씩 트레이닝 진행 - 평균 7시간씩 소요…
  • 코랩의 Free GPU, 12시간 제한에 유의할 것
  • Training에 실패하면 제일 먼저 데이터셋을 의심할 것
  • Fine tuning은 변경하며 실험

optimizer 종류, learning rate 값, weight decay 값, epoch 수, learning rate scheduling 방법, data augmentation 강도, 기타 regularizer 유무 등 변경해 보며 작업

학습이 모두 끝나게 되면 추출 스크립트를 통해서 Tensorflow2 기반의 모델을 다운로드 할 수 있음

Deploy the model

TF2 모델 추출 및 TFJS 모델로 변환

  1. 앞서 만든 TF2 모델을 TFJS용으로 변환 TensorflowJS converter package를 설치하면 간단히 변환 가능

  2. model.json과 weight 파일 여러 개가 만들어짐
    변환 결과로 모델의 그래프 구조에 대한 설명이 담긴 model.json 파일과, 가중치 매핑을 더 빠르게 하기 위해 가중치 파일은 ??되어 비동기로 불러올 수 있도록 여러 개로 제공됨

  3. 파일을 CDN에 업로드하면 준비 완료


CodeSandbox에서 테스트

변환한 모델이 잘 동작하는지 테스트해 보려면 model.json 파일과 weight 파일을 깃허브나 CDN에 올려놓고 코드샌드박스 같은 곳에서 동작을 테스트 해 보면 됨

간단한 이미지 검출기 프로젝트를 포크해서 만들고 깃허브에서 업로드한 모델 url을 로드그래프 모델 함수를 호출해서 모델이 잘 로드되는지 테스트해 봄

모델을 불러오는 데 성공했다면 테스트용 스크린샷 이미지를 업로드 하고 이걸 image tensor로 변환한 다음에 predict 함수로 이미지 안의 객체를 탐색함 (callback을 받을 수 있다면 일단 성공… 모델이 검출한 결과를 웹 브라우저를 통해 받아볼 수 있게 됨)

그런데 이 응답은 tensor의 배열이기 때문에 데이터를 해석해야 함. Tensor에서 데이터를 가져 올 때에는 동기/비동기 요청 api를 사용하면 되는데, 이 과정에서 saved_model_cli 라는 툴을 이용해서 각 tensor의 역할을 미리 파악할 수 있음

드디어 UI를 검출할 수 있는 맞춤법 검사기를 완성!
